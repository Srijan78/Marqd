1. Product Overview
1.1 Problem Statement
Sports organizations generate massive volumes of high-value digital media — match photographs, official highlight clips, broadcast stills — that scatter across global platforms within minutes of publication. This creates a critical visibility gap: proprietary content is widely misappropriated, redistributed without authorization, and monetized by third parties while rights owners have no reliable mechanism to detect, track, or act on these violations.
Track Objective
Develop a scalable, innovative solution to identify, track, and flag unauthorized use or misappropriation of official sports media across the internet. Enable organizations to proactively authenticate their digital assets and detect anomalies in content propagation in near real-time.
1.2 Solution — Marqd
Marqd is a three-layer content intelligence platform that addresses the problem end-to-end:

1.3 Key Differentiators
Proactive watermarking — ownership proof travels with every asset everywhere
Dual discovery engine — SerpApi covers the entire indexed web, YouTube Data API covers YouTube and Shorts
Video frame extraction — yt-dlp pulls keyframes from YouTube videos for watermark verification
Gemini AI domain classification — distinguishes authorized syndication from piracy
Auto-generated DMCA reports — legally actionable output in one click
Propagation anomaly detection — flags abnormal spread velocity and unexpected domains
Deployed entirely on Google Cloud — Cloud Run, Cloud SQL, Cloud Storage

2.2 Google AI Integration
The hackathon submission requires mandatory Google AI integration. Marqd integrates Gemini 3.1 Flash Lite in two meaningful, non-decorative ways:
DMCA Report Generation
When a violation is confirmed, violation metadata is sent to Gemini with a structured prompt. Gemini generates a professionally worded, legally formatted DMCA takedown notice — far superior to a hardcoded template.
prompt = f"""
Generate a formal DMCA takedown notice with the following details:
- Rights Owner: {org_name}
- Original Asset ID: {asset_id}
- Infringing URL: {violation_url}
- Detection Date: {date}
- Proof of Ownership: Watermark ID "{watermark_id}" extracted from infringing content
Format as a professional legal document.
"""
Domain Classification & Anomaly Detection
After SerpApi and YouTube Data API return results, those URLs are sent to Gemini for classification. Gemini categorizes each domain as: Authorized (official partners, news agencies), Suspicious (unknown blogs, aggregators), or Violation (piracy sites, unauthorized redistributors).
prompt = f"""
Classify each of these URLs for an official sports media asset.
Categories: AUTHORIZED | SUSPICIOUS | VIOLATION
Context: Asset belongs to {org_name}, authorized on {authorized_domains}
URLs: {url_list}
Return JSON: [{"url": "...", "category": "...", "reason": "..."}]
"""
2.3 YouTube Data API Integration
YouTube Data API v3 is used to actively search for new uploads containing stolen sports content. Unlike SerpApi which finds indexed web pages, YouTube API searches inside YouTube's own video database — covering YouTube videos, Shorts, and newly uploaded content before Google indexes them.
How It Works
Each registered asset has associated search keywords (e.g. team name, match name, player name, event)
Every 6 hours, the scanner searches those keywords via YouTube Data API
New videos matching those keywords are returned with video IDs and metadata
yt-dlp downloads keyframes from suspicious videos at low resolution
Watermark decoder runs on extracted frames — if asset ID found, confirmed violation
from googleapiclient.discovery import build
youtube = build('youtube', 'v3', developerKey=YOUTUBE_API_KEY)
# Search for new uploads matching asset keywords
response = youtube.search().list(
    q=asset_keywords,
    part='snippet',
    type='video',
    order='date',              # Most recent first
    publishedAfter=last_scan_timestamp,  # Only new uploads
    maxResults=25
).execute()
# Extract frames using yt-dlp and decode watermark
# yt-dlp --extract-frames --frame-interval 30 {video_url}
YouTube API Quota Usage
YouTube Data API v3 gives 10,000 free units per day. A search request costs 100 units, video detail lookup costs 1 unit. At 100 search requests per day you can cover 100 keyword searches — more than enough for a focused set of registered assets.

3. Core Features
3.1 Asset Registration & Watermarking
Sports org uploads official media through the dashboard. Before the asset is stored or distributed, Marqd embeds an invisible unique watermark using the blind_watermark library (DWT+DCT frequency-domain embedding). The watermark carries a unique Asset ID (e.g., IPL_2025_FINAL_IMG_047) that is stored in the database alongside asset metadata.
Supports images (JPG, PNG, WebP) and video (MP4, MOV)
For video: watermark embedded per-frame using video-invisible-watermark
Org assigns search keywords per asset for YouTube monitoring
Watermarked version uploaded to Cloud Storage for use as SerpApi scan URL
Original unwatermarked asset stored separately
pHash also generated for each asset as secondary fingerprint
3.2 Dual Discovery Engine
Every 6 hours, a background scheduler (APScheduler) runs a scan job using two parallel discovery mechanisms that complement each other to maximize coverage across the internet.
Discovery Channel 1 — SerpApi Reverse Image Search
For each registered asset, the system calls SerpApi's Google Reverse Image Search endpoint with the watermarked asset's Cloud Storage URL. SerpApi queries Google's entire indexed web and returns every URL where that image or visually similar content appears — covering news sites, blogs, sports forums, Reddit, public social pages, piracy streaming sites, and more.
Discovery Channel 2 — YouTube Data API
For each registered asset's keywords, the system searches YouTube Data API for newly uploaded videos. Videos published after the last scan timestamp are retrieved, frames are extracted using yt-dlp, and the watermark decoder runs on those frames. This catches stolen content uploaded to YouTube and YouTube Shorts even before Google indexes the video page.

3.3 Watermark Verification
For every URL flagged by either discovery channel, Marqd runs watermark extraction to confirm ownership before raising a violation. This eliminates false positives and provides cryptographic proof of ownership.
SerpApi result: download image from flagged URL, run blind_watermark decoder
YouTube result: extract keyframes via yt-dlp, run decoder on each frame
If watermark Asset ID matches database record — confirmed violation
pHash comparison runs as secondary check for modified images
Confidence score calculated from watermark strength + pHash distance
3.4 Propagation Intelligence
Beyond simple violation detection, Marqd maps how each asset spreads across the web over time. This directly addresses the track requirement to detect anomalies in content propagation in near real-time.
Propagation timeline — first seen date per domain across both discovery channels
Spread velocity calculation — how fast the asset is appearing on new domains
Geographic distribution — which regions the asset is appearing in
Platform breakdown — web vs YouTube vs other platforms
Anomaly flags — content on piracy domains, spread faster than baseline, unexpected regions
World map visualization showing asset spread with violation markers
3.5 DMCA Report Generation
When a confirmed violation is found, the org can generate a DMCA takedown notice in one click. Gemini 3.1 Flash Lite generates the professional legal language. reportlab compiles it into a downloadable PDF containing asset details, violation evidence, watermark extraction proof, and takedown request language appropriate for the platform (web host vs YouTube).

4.4 Database Schema
assets table
CREATE TABLE assets (
  id                UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  org_id            VARCHAR(100) NOT NULL,
  asset_name        VARCHAR(255) NOT NULL,
  asset_type        VARCHAR(20) NOT NULL,        -- image | video
  watermark_id      VARCHAR(100) UNIQUE NOT NULL,
  phash             VARCHAR(64),
  storage_url       TEXT NOT NULL,               -- Cloud Storage URL (watermarked)
  original_url      TEXT,                        -- Cloud Storage URL (original)
  search_keywords   TEXT[],                      -- keywords for YouTube API search
  authorized_domains TEXT[],                     -- whitelist per asset
  created_at        TIMESTAMP DEFAULT NOW()
);
violations table
CREATE TABLE violations (
  id                UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  asset_id          UUID REFERENCES assets(id),
  violation_url     TEXT NOT NULL,
  domain            VARCHAR(255),
  platform          VARCHAR(20),                 -- WEB | YOUTUBE
  youtube_video_id  VARCHAR(50),                 -- populated for YouTube violations
  domain_category   VARCHAR(20),                 -- AUTHORIZED | SUSPICIOUS | VIOLATION
  watermark_match   BOOLEAN DEFAULT FALSE,       -- confirmed via extraction
  confidence        FLOAT,                       -- 0.0 to 1.0
  gemini_reason     TEXT,
  detected_at       TIMESTAMP DEFAULT NOW(),
  dmca_generated    BOOLEAN DEFAULT FALSE
);
scan_logs table
CREATE TABLE scan_logs (
  id                UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  asset_id          UUID REFERENCES assets(id),
  scan_type         VARCHAR(20),                 -- scheduled | manual
  serpapi_results   INTEGER DEFAULT 0,           -- URLs found via SerpApi
  youtube_results   INTEGER DEFAULT 0,           -- videos found via YouTube API
  violations        INTEGER DEFAULT 0,
  serpapi_credits   INTEGER DEFAULT 0,           -- SerpApi credits used
  youtube_units     INTEGER DEFAULT 0,           -- YouTube API units used
  started_at        TIMESTAMP DEFAULT NOW(),
  completed_at      TIMESTAMP
);

8. Demo Flow
The demo tells one clear story: official sports content gets stolen across two platforms simultaneously — our system finds both violations, proves ownership on both, and generates takedown notices in one click.
Step 1 — Register Asset
Upload an official IPL match photo. Assign keywords: 'IPL 2025 Final Rohit Sharma'. System watermarks it invisibly. Show original vs watermarked side-by-side — visually identical. First wow moment.
Step 2 — Simulate Theft (Pre-Demo Setup)
Web violation: upload the watermarked image to Imgur with a public URL
YouTube violation: upload a short clip containing the watermarked image to a test YouTube account with title 'IPL 2025 insane moment highlights'
Step 3 — Run Scan
Hit Scan Now on dashboard. Both discovery channels run simultaneously. SerpApi finds the Imgur URL. YouTube API finds the test video by keyword. Both results appear in the violations panel within seconds.
Step 4 — Watermark Verification
System downloads the Imgur image and extracts watermark — Asset ID confirmed. System downloads a keyframe from the YouTube video via yt-dlp and extracts watermark — Asset ID confirmed again. Two confirmed violations, two platforms, one scan.
Step 5 — Propagation Map
Map updates showing violations on two platforms. Anomaly flagged — content appeared on an unknown domain and an unverified YouTube channel within hours of registration.
Step 6 — DMCA Reports
One click generates two DMCA notices — one worded for a web host takedown, one worded for a YouTube copyright strike. Both PDFs download instantly with watermark extraction as legal proof.
9. Known Limitations & Judge Responses
Limitation
Honest Answer for Judges
Instagram Reels and TikTok not covered
Instagram and TikTok provide no public API for content search — this is an industry-wide limitation affecting enterprise tools equally. Our architecture supports integration the moment those APIs become available.
YouTube keyword search can be evaded by obscure titles
Anyone trying to monetize stolen sports content needs discoverability — they cannot avoid relevant keywords entirely. Completely obfuscated uploads get no views and represent minimal revenue loss.
SerpApi 250 searches/month on free tier
Production deployment uses paid tier. Scheduled monitoring every 6 hours is architecturally identical to how enterprise tools like Copytrack and Digimarc operate.
Watermark can degrade under heavy compression
blind_watermark DWT+DCT is robust to typical social media re-encoding, resizing, and minor crops. Heavy adversarial attacks are a known research problem across all watermarking systems.
Demo uses planted violations
Every hackathon demo uses controlled data. The detection pipeline is identical regardless of who uploaded the content. Both discovery channels are fully functional.
